# 데이터 안내

이 디렉터리는 서울시 공공자전거 수요예측 프로젝트에서 사용하는 원천 데이터와 전처리 결과를 관리합니다.

## 1. 디렉터리 구성

```text
data/
├── README.md
├── raw/
│   ├── bike_usage/
│   ├── station/
│   └── station_history/
└── processed/
    ├── README.md
    ├── station_cluster_map.csv
    └── cluster_hourly_demand.csv
```

## 2. 원천 데이터

### 2.1 공공자전거 이용정보

* 데이터: 서울특별시 공공자전거 이용정보(시간대별)
* 분석 기간: 2023년 1월 1일 ~ 2024년 12월 31일
* 주요 정보: 대여일자, 대여시간, 대여소 식별자, 이용건수
* 저장 위치: `data/raw/bike_usage/`

월별 원천 CSV 파일은 파일 크기가 크기 때문에 GitHub 저장소에 포함하지 않습니다.

### 2.2 공공자전거 대여소 정보

* 데이터: 서울시 공공자전거 대여소 위치정보
* 주요 정보: 대여소 식별자, 대여소명, 위도, 경도
* 저장 위치:

  * `data/raw/station/`
  * `data/raw/station_history/`

분석 기간 중 대여소 정보의 변경 가능성을 고려하여 2023년과 2024년의 시점별 대여소 자료를 함께 점검했습니다.

### 2.3 기상 데이터

현재 최종 모델에는 기온, 강수량, 풍속 등의 기상 변수를 사용하지 않았습니다.

미래 시점의 관측 날씨를 사용하는 정보 누출을 피하고, 모든 비교 모델을 동일한 조건에서 평가하기 위해 기본 모델 비교에서는 기상 변수를 제외했습니다.

`statsmodels`의 `SARIMAX` 클래스를 사용해 모델을 구현했지만 외생변수를 입력하지 않았으므로, 실제 모델 구조는 SARIMA입니다.

## 3. 원천 데이터가 GitHub에 없는 이유

공공자전거 월별 이용정보는 개별 파일의 크기가 수백 MB에 이를 수 있습니다. 따라서 다음 이유로 원천 파일을 저장소에서 제외합니다.

1. GitHub 저장소 용량을 불필요하게 증가시키지 않기 위해
2. 공개 데이터의 원본을 중복 배포하지 않기 위해
3. 분석 코드와 핵심 결과 중심으로 저장소를 구성하기 위해

원천 데이터는 서울시 공공데이터에서 내려받아 각 하위 폴더에 배치해야 합니다.

## 4. 최종 가공 데이터

GitHub 저장소에는 다음 두 개의 최종 가공 데이터를 포함합니다.

### `station_cluster_map.csv`

GPS 좌표를 기준으로 K-Means 군집화를 수행한 대여소별 군집 배정 결과입니다.

주요 정보:

* 대여소 식별자
* 대여소명
* 위도
* 경도
* `cluster_id`

### `cluster_hourly_demand.csv`

군집별 시간당 공공자전거 대여 수요 데이터입니다.

주요 정보:

* 시간
* `cluster_id`
* 시간당 대여건수 `rental_count`
* 시간 관련 파생 변수

최종 데이터의 분석 단위는 개별 대여소가 아니라 다음 조합입니다.

```text
GPS 기반 군집 × 1시간
```

## 5. 최종 데이터 규모

* 분석 기간: 2023-01-01 ~ 2024-12-31
* 전체 시간대: 17,544개
* 군집 수: 3개
* 전체 행 수: 52,632행
* 총 대여량: 88,708,877건
* 예측 타깃: 군집별 시간당 `rental_count`

## 6. 데이터 생성 순서

최종 가공 데이터는 다음 노트북을 순서대로 실행하여 생성합니다.

```

모델 학습과 평가는 다음 노트북에서 수행합니다.

```
